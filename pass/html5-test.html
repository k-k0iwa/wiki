Webの基礎

HTTPのリクエストメッセージ

GETメソッドでリクエストパラメータが指定された場合、リクエストパラメータはリクエストURLに付加されます。「?」以降の文字列がクエリ文字列と呼ばれ、リクエストパラメータの内容を表します。

POSTメソッドでリクエストパラメータが指定された場合は、メッセージボディに付加されます。

HTTP cookie

  • 元来HTTPはステートレスなプロトコルですが、クッキーを利用することでショッピングサイトなどのようにステートフルなサービスを可能にします。
  • クッキーの情報はHTTPヘッダに含んで通信を行うので基本的にJavascriptでアクセスできますが、設定によりアクセス出来ない場合があります。
  • クッキーはクライアント（ブラウザ）に保存され、ユーザはブラウザの機能を利用することで履歴の削除などと同時に削除することができます。
  • クッキーは1つ4KB、1サーバーに20個、トータル300個まで保存できますが、制限があるためHTML5では、Web Storageなど大容量を扱えるAPIも用意されています。
  • クッキーには有効期限を設定することができ、有効期限を超えると保存されていたクッキーは無効になります。有効期限が指定されていると、有効期限内はブラウザを終了しても保存されていますが、指定されていない場合ブラウザの終了とともに削除されます。

HTTPヘッダフィールド

HTTPヘッダフィールドの内、Webアクセスの高速化を目的に設定できるもの

  • Cache-Control（Cache を制御）
  • Expires（ページの有効期限以前であれば、キャッシュを利用させる）
  • Last-Modified（コンテンツが前回アクセス時から更新されていない場合、キャッシュを利用する）

BASIC認証、Digest認証

BASIC認証では、ユーザが入力したユーザ名とパスワードは一切暗号化されることなく送信されます。

Digest認証ユーザ名とパスワードは、MD5というハッシュ関数を使用して、ユーザ名とパスワードをダイジェスト(ハッシュ)化して送信します。

HTML5の文書型宣言

HTML5の文書型宣言の基本形は、「<!DOCTYPE」＋1個以上のスペース＋「HTML」＋0個以上のスペース＋「>」です。アルファベットは大文字でも小文字でもかまいません。

title要素

title要素自体は条件によっては省略可能ですが、開始タグ・終了タグともにタグの省略は一切できません。

文字（実体）参照

名称を指定する以外にも、文字コード(番号)を指定する方法もあります。
文字コードを指定する場合には2種類の書き方があり、&#10進数; と、&#x16進数; です。

HTTPS

HTTPSを使用して、Webサーバの正当性を確認するために必要なものは2つある。

  • 認証局
  • サーバ証明書

Base64

  • バイナリデータを印字可能文字に置き換える仕組みである
  • 英数字と記号を使用する
  • 端数に当たる部分には'='を使用する
  • 変換後は1行64文字で改行される

XHTML書式の特徴

  • html要素の開始タグにXHTML用の名前空間を指定する
  • 要素名や属性名は全て小文字で記述する必要があります
  • 属性の値は「"」や「'」で囲む必要があります
  • 空要素には終了タグを付加するか、開始タグの閉じかっこを「/>」としなければならない

HTML

空要素のNG記述

  • 「/」と「>」の間には空白文字は入れられない
  • 終了タグを指定できない
  • <br><br />のように記述されていた場合、空要素が2つ連続して配置されていると認識され、文法エラーにはならない

source要素

  • picture要素内のsource要素には、media属性またはtype属性のいずれかの指定が必要
  • picture要素内のsource要素には、srcset属性が必須
  • video要素とaudio要素にはsrc属性が必須

script要素とnoscript要素

  • HTML5.1からは、script要素にnonce属性が指定できるようになりました。
  • script要素にデータブロックを指定する場合、type属性の指定は必須となり、src属性は使用できなくなります。
  • noscript要素は、head要素内にも配置できますが、XHTML構文では使用できません。

textarea要素

  • textarea要素のサイズはデフォルトで20文字×2行で、rows,colsにより変更が可能である
  • resizeプロパティの値には、「none：リサイズを許可しない」、「horizontal：水平方向のみリサイズを許可する」、「vertical：垂直方向のみリサイズを許可する」、「both：水平・垂直方向ともリサイズを許可する。（デフォルト）」、「inherit：親要素の値を継承する」がある。
    無効な値を指定した場合、デフォルトと同じ動きとなります。

main要素

  • role属性を併用することが推奨されている
  • WHATWGとW3Cで仕様に違いがある

object要素

画像、音声、HTML文書など様々な外部リソースを文書内に取り込む際に使用します。ファイルの指定はdata属性を使用します。下記はサンプルです。


<object data="sample.mpg" …>

details要素

<details>タグで囲われた部分を非表示にし、必要に応じて利用者がクリックすることで参照できるようにできます。summary要素は、details要素の子要素で、省略時に表示される「詳細」（ブラウザにより異なる）を別の文字列に置き換える。

videoタグ

動画圧縮規格がh.264の映像を表示する場合、sourceタグに設定する適切なtype属性はvideo/mp4になります。

role属性

既存の要素の役割・意味を上書きして変更する

mark要素

一般的なブラウザでは蛍光ペンでマークしたような表示となります。オリジナルでは通常のテキストである部分を、そのWebページの読者に注目してもらうためにハイライト表示させるのがmark要素の役割です。
また、検索結果の表示における検索語もmark要素を使用します。

開始タグと終了タグの両方を省略できる要素

条件によって、開始タグと終了タグを省略できる要素は以下の5つ！！！

  • <html>
  • <head>
  • <body>
  • <tbody>
  • <colgroup>

コメント

「<!--」と「-->」は続けて書く必要があります。コメント内部のテキストは「>」または「->」で開始することはできず、「-」で終了できません。また、コメント内部のテキストに「--」を含むこともできません。

コンテンツモデルがトランスペアレントの要素

該当の要素は以下の9つ！！！

  • a要素
  • ins要素
  • del要素
  • object要素
  • audio要素
  • video要素
  • canvas要素
  • map要素
  • noscript要素

id属性、class属性の記述ルール

id属性

id属性の値には空白文字を含むことはできません。また、値には必ず1文字以上入れる必要があります。

class属性

class属性の値には空白文字で区切って複数指定できます。空白文字は、連続して複数個を入れても問題ありません。また、属性の「=」の前後には空白文字を入れることができます。

base要素、title要素のルール

base要素

head要素内にbase要素は1つまでしか記述できない。また、省略可能です。

title要素

通常head要素内にtitle要素を1つ入れる必要があるが、条件によっては省略することが可能です。

セクション

セクションをあらわす要素は以下の4つです。
セクショニング･ルートとなる要素はセクショニングコンテンツに含まれません。

  • section要素
  • article要素
  • aside要素
  • nav要素

セクショニング･ルート

セクションをあらわす要素は以下の6つです。

  • body要素
  • blockquote要素
  • fieldset要素
  • figure要素
  • td要素
  • details要素

em要素とstrong要素の違い

em要素はタグをつける位置を変えると文章の意味も変わるが、strong要素はタグをつける位置が変わっても文章の意味に影響を与えない。

img要素の属性について

  • src属性は必須
  • alt属性は条件によって省略可能
  • width属性、height属性には「CSSピクセル数（単位の無い0以上の整数）」しか指定できません

hr要素

「段落レベル」での主題の区切りをあらわします。

dl要素

dl要素内は、空、もしくは1つ以上のdt要素と1つ以上のdd要素を記述できます。

CSS

背景画像の指定

  • CSS3からは、1つのボックスに複数の背景画像を指定できるようになりました
  • 背景画像をカンマ区切りで指定すると、先（左側）に指定した背景画像ほど上（画面上で手前）に重なって表示されます。
  • 背景色はすべての背景レイヤーのうち、一番下のレイヤーにのみ表示されます。そのため、背景色は一番下の背景レイヤーとなる一番右側の値の1つとして指定することになっています。結果として背景色は値を1つしか指定できません。

text-transform

  • lowercase：テキストを全て小文字に変換します。
  • uppercase：テキストを全て大文字に変換します。
  • capitalize：単語の1文字目を大文字にします。

white-space

pre-wrapはブラウザで表示させる際にもソースコード上の改行位置で改行させ、連続する半角スペース・タブ・改行コードを1つの半角スペースにまとめず、テキストがウィンドウの右端まで来たときには自動的に行を折り返すようにしたい場合に指定する値

@importの書式

  • @importのあとに外部スタイルシートのURLを文字列として（"または'が必要）指定する
  • 関数形式で指定する（"や'は省略可能）
  • URLのあとに、メディア型やメディアクエリを指定することができる

1つ目の使用例


@import "style.css";

2つ目の使用例


@import url("style.css");

3つ目の使用例


@import "style.css" screen, print;

hyphensプロパティに指定できる値

指定できる値は「manual」「none」「auto」の3つのみ。
「manual」を指定すると、ハイフネーションは&shy;が入力されている箇所でのみ行なわれます。
「auto」を指定すると、ブラウザが適切な箇所でハイフネーションを行ないます。（lang属性による言語の指定が必要となります。）

box-shadowプロパティの値の順番

1番目から、「左右にずらす」「上下にずらす」「ぼかす範囲」「拡張させて大きくする距離」となる。
色の値とキーワードのinsetは、数値のグループの前後どちらかに順不同で指定できます。

マルチカラムレイアウト

1つのボックスの内部を複数の段に分割するタイプのレイアウト

column-count
何段組にするのかを設定。初期値：auto
column-width
段の幅を設定。初期値：auto
columns
column-countとcolumn-widthの値をまとめて指定できる
column-gap
段と段の間隔を設定。初期値：normal
column-rule
線種(column-rule-style)・色(column-rule-color)・太さ(column-rule-width)を個別に順不同で設定。
column-span
段組が設定されているボックスの幅いっぱいに（すべての段をまたいで）表示させる。初期値：none

メリット・デメリット

メリット

  • 画面サイズさえあえば、現在存在しないデバイスにも対応可能
  • URLが同じとなるためSEOに有利
  • サイトの読み込み時間を短縮できる
  • HTMLが1つですむのでコストの削減につながる可能性がある

デメリット

  • 設計の難易度が高く、逆にコストが高くなることも
  • モバイル向けには画像や動画を切り替える必要があり難しい
  • 画面フローの変更には対応できない
  • HTML/CSSのサイズが増加し、下手をするとページ表示や動作が遅くなる

resolution（デバイスの解像度）

メディアクエリで解像度を指定する場合の指定できる単位については以下の3つがあります。

  • dpi1インチあたりドット数
  • dpcm1センチあたりドット数
  • dppx1ピクセルあたりドット数

高解像度向け対応

「デバイスピクセル」はデバイスとして物理的にもっている。「CSSピクセル」はCSSの中で理論的に解釈される。

Retinaディスプレイでは、4デバイスピクセルが1CSSピクセルとして扱われます。Retina対応の方法は大きく下記の4点あります。

  • 大きなサイズの画像を使用
    実際に表示させたいサイズの倍の大きさの画像を用意し、表示サイズをimg要素で指定すれば、高解像度の画像を表示させることが可能です。
  • メディアクエリを使用
    background-imageで表示するCSSを用意し、それらをメディアクエリで切り替えることで、表示先が高解像度の場合は高解像度の画像を表示させることが可能です。
    「2デバイスピクセル：1CSSピクセルの比率以上のデバイスに対しては、倍サイズの画像であるimg-2x.pngを使用する」という場合には以下のように指定します。（-webkit-min-device-pixel-ratioとmin-resolutionを使用します）
    
.img {
	background-image: url(img.png);
}
@media (-webkit-min-device-pixel-ratio: 2), (min-resolution: 2dppx) {
	.img {
		background-image: url(img-2x.png);
	}
}
  • SVGやWebフォントの使用
    拡大によりぼやける問題が発生しないため
  • JavaScriptライブラリの使用
    解像度の比率に応じて画像を入れ替えるJavaScriptライブラリがあるため、それを利用する。

API

JavaScriptを使って描画するグラフィックスの特徴

  • 動くグラフィックスを描画できる
  • HTMLのcanvas要素を利用できる
  • 2Dグラフィックスを利用できる
  • SVGを利用すると、拡大・縮小しても粗くならない画像が描画できる

ブラウザにデータを保存するAPI

Web Storage

  • キーと値の組み合わせでブラウザにデータを蓄積する
  • アプリケーションのオフライン利用や高速化が期待できる
  • セッションストレージ（ウィンドウを閉じるとデータが消失）とローカルストレージ（ウィンドウを閉じてもデータが消えない）の2種類ある

Indexed Database API

  • キーと値の組み合わせでJavaScriptのオブジェクトを蓄積し利用する
  • インデックスやトランザクション（データベースの一連の処理）を利用できる
  • 大量のデータを扱うオフライン利用可能なアプリケーションのデータベース

Web Workers

  • ブラウザでスクリプト処理をバックグラウンドで実行する
  • JavaScriptで複数のスレッドを走らせ、並列処理を行なうことができる
