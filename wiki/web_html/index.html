DOCTYPE宣言

  • HTML5には実際DOCTYPE宣言は不要だが、ブラウザの表示モードを「標準モード」にする目的で指定することになっている
  • 基本となるDOCTYPE宣言は<!DOCTYPE html>です。DOCTYPE宣言は大文字で書いても小文字で書いても良い

補足説明

HTML5であってもHTML4.01やXHTML1.0などのDOCTYPE宣言がそのまま使用できる。

HTML5で使用が許可されているDOCTYPE宣言


<!DOCTYPE html>
<!DOCTYPE html SYSTEM "about:legacy-compact">
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.0//EN">
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.0//EN" "http://www.w3.org/TR/REC-html40/strict.dtd">
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN">
<!DOCTYPE html PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" "http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

要素とカテゴリー

HTML5よりも前のHTML/XHTMLでは、要素は大きくインライン要素とブロック要素の2種類に分類されていましたが、このインラインかブロックレベルかという要素の大きな分類は廃止され、要素は7種類のカテゴリーに定義されています。

以前はブロックレベル要素以外の要素をbody直下に配置することは基本的にできませんでした。しかし、HTML5ではブロックレベルかどうかという分類はなくなり、従来のインライン要素に該当する要素でも、body直下に配置できるようになっている。

class属性

それを指定した要素が属する種類・分類を示すための属性。名前は空白文字で区切って複数指定可

id属性

それを指定した要素に固有の名前をつけるための属性。ページ内の他のid属性に同じ値を指定することはできない。値の中に空白文字を含むことはできない。

lang属性

言語の種類を示す属性。値には、BCP47という仕様で定義されている言語タグを使用する。

属性値 言語
ja 日本語
ja-jp 日本における日本語
en 英語
en-au オーストラリアにおける英語
de ドイツ語

title属性

その要素に関連する補助的な案内や助言的な情報を提供するための属性。一般的なパソコン環境のブラウザでは、ツールチップで表示される。

dir属性

要素内のテキストを表記する方向を示す属性。「ltr」は「left-to-right（左から右へ）」の略で、「rtl」は「right-to-left（右から左へ）」の略です。

tabindex属性

フォーカスの可・不可を制御し、フォーカスを可能にした場合には「Tab」キーによる移動の順序も設定可能な属性。値には整数が指定でき、0以上を指定するとフォーカスされるようになり、負の値を指定するとフォーカスされなくなる。「Tab」キーによる移動の順序を設定するには、1以上の整数を指定します。（ソースコード上の順番に関係なく、値の小さいものから大きいものへと移動するようになります）。値に0を指定すると、ソースコード上に登場する自然な順序のままフォーカスされるようになる。

どの要素であってもこの属性が指定されるとインタラクティブコンテンツになります。

dropzone属性

この要素にドロップされたデータがどうなるかを示す属性。値には「copy（コピーされる）」「move（移動する）」「link（リンクが作成される）」のいずれか1つずつが指定できます。
さらに空白で区切って「受け入れ可能な文字列またはファイルの種類」を指定することもできる。その場合は、文字列なら「string:MIMEタイプ（例：string:text/plain）」、ファイルなら「file:MIMEタイプ（例：file:image/jpeg）」の書式で指定します。

カスタムデータ（data-*）属性

使用するのに適当な属性や要素がない場合に、任意の要素に独自の属性を追加できるように用意されたもの。値は自由に設定でき、1つの要素にいくつでも指定できる。属性は必ず「data-」という文字列で開始し、そのあとに少なくとも1文字以上を続けなければなりません。また、属性名の中にアルファベットの大文字を含むことはできません。

WAI-ARIA関連の属性

HTML5では、WAI-ARIA（Accessible Rich Internet Applications 1.0）という別の仕様で定義されているrole属性およびaria-*属性を任意の要素に対して指定することができます。

role属性
要素の役割が何であるのかを示す属性。値には、「application」「banner」「search」「navigation」「tab」「tabpanel」などが指定できる。
aria-*属性
要素のその時点での状態や特性などを示すために使用できる属性で、「aria-hoge」といった形式の属性名を持ちます。
例：「aria-checked」属性･･･チェックボックスなどがチェックされている状態かどうかを示す。値は「true」「false」「mixed」。

用語解説

WAI-ARIA

  • 2014年にW3C勧告として公開された、動的コンテンツや複雑なユーザーインターフェースを持つサイト（Webアプリケーション）をアクセシブルにするための仕様である「Accessible Rich Internet Applications 1.0」の略称。
  • 「WAI」はWeb Accessibility Initiativeの略で、Webアクセシビリティの向上を目的としたガイドラインや仕様を策定するW3C内の組織を示し、「ARIA」はAccessible Rich Internet Applicationsの略です。

W3C

  • W3C（World Wide Web Consortium）は、Web関連技術の標準化と推進を目的とした国際的な非営利団体で、HTML・XHTML・XML・CSS・DOM・WAI-ARIAなどの仕様を策定している。

Webアクセシビリティ

  • 「心身に機能的な制限があってもWeb上の情報にアクセスでき、理解可能な状態で欠落のない情報が得られ、操作も可能であること」を指す。
  • Webアクセシビリティが確保された状態のことを「アクセシブル」と表現する。

全体構造とメタ情報

html要素

HTML文書のルート（DOMツリーの頂点／要素全体の先祖）となる要素です。
html要素には、グローバル属性であるlang属性を指定してその文書で使用されている言語の種類を示すことが推奨されている。

head要素

HTML文書のメタデータコンテンツを入れるための要素です。
通常のHTML文書の場合、head要素内にはtitle要素を1つ入れる必要があります。base要素は任意ですが、1つまでしかいれられません。

body要素

HTML文書のコンテンツを入れるための要素です。

title要素

HTML文書のタイトルまたは名前であることを示す要素です。1つのHTML要素につき1つしか配置できません。

meta要素

HTML文書自身に関する情報を指定できる空要素。

指定できる属性
属性名 説明 例
charset 文字コード
UTF-8、Shift_JIS、EUC-JPなど		 <meta charset="utf-8">
name メタデータの名前
「description」文書の説明、「keywords」キーワード、「author」文書の作者など		 <meta name="description" content="説明">
http-equiv プラグマディレクティブ（プラグマ指示子）。HTML文書の状態や挙動を指示する命令のこと。
「X-UA-Compatible」IEの互換表示設定の指示、「refresh」リダイレクト・再読み込みの指定など		 

<meta http-equiv="X-UA-Compatible" content="IE=Edge">

<meta http-equiv="refresh" content="10; url=page2.html">
content メタ情報の値。テキストを入力できる。

base要素

HTML文書内で指定されている相対URLの基準にするURLを設定するための空要素です。必ずhead要素内で指定します。target要素を使用することでデフォルトの表示先（ブラウジングコンテキスト）を設定しておくこともできます。指定できる属性はhref属性とtarget属性のみ。

用語解説

ブラウジングコンテキスト

HTML5の文書では、HTML文書を表示させるウィンドウやタブ、インライン･フレームなどのことをブラウジングテキストと呼んでいます。

セクションと基本構造

セクショニングコンテンツに分類される4つの要素（article要素・section要素・aside要素・nav要素）はそれぞれ、その範囲が1つのセクションであることを示します。HTMLのセクションとは、見出しとそれに対応する文章全体を含む範囲のことで、長い文章における「章」や「節」、新聞やブログなどの「1つの記事」などがこれにあたります。

見出しとセクショニングコンテンツは常に1対1のセットで使用する決まりになっているわけではなく、「見出しのないセクショニングコンテンツ」も存在すれば「セクショニングコンテンツのない見出し」も存在します。
これらのうち、「見出しのないセクショニングコンテンツ」はそのまま見出しのないセクショニングコンテンツとなりますが、「セクショニングコンテンツのない見出し」について内部的にはセクションが存在するものとして認識される点に注意してください。
つまり、見出しがあるだけでそこにはセクションが存在することになるということです。このような実際のタグを伴わないセクションは暗黙のセクションと呼ばれる。

article要素

新聞や雑誌の記事などそれだけで全部の／それだけで完結しているセクションをあらわす場合に使用します。ブログの記事や学術論文などにも使用できます。

section要素

一般的なセクションを表す要素です。HTML5でセクションをあらわす要素は全部で4つありますが、他の3つのセクション要素は特定の用途向けであり、それらに該当しない普通のセクションをマークアップする場合に使用します。

補足説明

article要素とsection要素の使い分け

ひとつのまとまったコンテンツの全体を含んでいるならarticle要素、全体の一部であればsection要素を使用します。

aside要素

その内容がそれを含んでいるセクションの内容とは関係が薄く、別扱いにした方がよさそうなセクションをマークアップする場合に使用します。
たとえば、広告・補足記事・プル・クォート（本文の一部を抜粋して目立つように掲載し、読者の興味を引かせる部分）、nav要素のグループ、その他のメインコンテンツとは分けたほうがよさそうな各種コンテンツも使用できます。

nav要素

そのページにおけるナビゲーションのリンクを含むセクションをあらわす場合に使用します。主要なナビゲーションだけに使用し、必要でなければフッター領域にあるリンクなどには基本的に使用しないでください。

header要素

自分自身を含むもっとも近いセクションまたはセクショニング・ルートのヘッダーであることを示す要素です。
一般的に、header要素の内容としては見出し・ナビゲーション、ロゴ、検索フォーム・目次などを含みます。通常は見出しを含みますが必須ではありません。

footer要素

自分自身を含むもっとも近いセクションまたはセクショニング・ルートのフッターであることを示す要素です。
footer要素の内容としては、著作権情報・お問い合わせ先（address要素）・関連文書へのリンク（「前ページ」、「次ページ」など）・記事の執筆者名などを含みます。ただし、執筆者名に関してはheader要素に入れても問題ありません。

main要素

その範囲がページ全体におけるメインコンテンツであることを示す要素です。
そのページに特有のコンテンツのみを入れ、サイト内の複数のページで共通しているナビゲーションやロゴ、検索フォーム、著作権情報などのコンテンツは含めないようにしてください。

address要素

自分自身を含むもっとも近いarticle要素またはbody要素の内容に関する問い合わせ先を示す要素です。articleに含まれていない場合は、body要素（ページ全体）に関する問い合わせ先を示していることになります。

※address要素の内容として入れられるのは、問い合わせ先の情報だけです。著作権情報や更新日のような問い合わせ先以外の情報は入れられませんので注意してください。また、住所であってもそれがarticle要素またはbody要素の内容に関する問い合わせ先ではない場合には使用できません。問い合わせ先ではない普通の住所には、p要素を使用します。

セクショニング・ルート

その範囲が外部とは別の独立したセクションをもつ「セクションのルート要素」となり、内容として含んでいる見出しやセクショニングコンテンツが外部のアウトラインに一切影響を与えなくなる要素をセクショニング・ルートという。

セクショニング・ルートに分類される要素

  • bodyHTML文書のコンテンツ全体を入れる要素
  • blockquote引用されたコンテンツ（ブロックレベル）
  • fieldsetフォーム部品のグループ
  • figure図版
  • td表のセル（データ用）
  • detailsディスクロジャーウィジット（詳細折りたたみ要素）

テキスト関連の要素

a要素

a要素にhref属性を指定すると、リンク（ハイパーリンク）になります。href属性は必須ではありませんが、href属性を指定していない状態ではそれ以外の属性も指定できません。（グローバル属性は指定できます。）

a要素のコンテンツモデルはトランスペアレントです。したがって、HTML5のa要素は旧分類でのブロックレベル要素も要素内容として含むことができます。

em要素

強調を示す要素。emを入れ子にすることで、強調の度合いを強くすることができます。

strong要素

「重要」「重大」「緊急」「深刻」のいずれかであることを示す要素。strongを入れ子にすることで、意味の度合いを強くすることができます。

blockquote要素

引用してきた文章であることを示すブロックレベル要素。（インライン要素の場合はq要素を使用する。）

必要に応じて、要素内にfooter要素やcite要素を配置して引用文に関する情報（題名や作者名、参照先へのリンクなど）を加えることが可能です。
引用した文章がWeb上のものであれば、cite属性の値として引用元のURLを指定することができます。

※引用文はオリジナルの文章と完全に一致している必要はありません。

q要素

引用してきた文章であることを示すインライン要素。（ブロックレベル要素の場合はblockquote要素を使用する。）q要素はインライン要素として引用文を示すときに使用します。

引用した文章がWeb上のものであれば、cite属性の値として引用元のURLを指定することができます。

※q要素にはブラウザが引用符をつけて表示することになっていますので、この要素を使用する場合は引用文の前後に引用符はつけないでください。

cite要素

創作物の出典・出所をあらわすための要素。要素内容には、作品の「タイトル」「作者名」「参照先のURL」「参照先を示す略語」のいずれかを含めなければいけません。

mark要素

元々はそうなっていないテキストの一部を、参照してもらいやすいように目立たせた部分であることを示す。
